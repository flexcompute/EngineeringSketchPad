The following list of attributes drives the ZAero geometric definition.

  • capsLength This attribute defines the length units that the *.csm file is generated in. ZAero grids MUST be in units of meter, as such the geometry is scaled accordingly based on this value.
  • capsReferenceArea [Optional] This attribute may exist on any Body. Its value will be used as the reference area in ZAero's input file with its units assumed to be consistent with the attribute "capsLength". No conversion takes place if "capsLength" isn't set. This value may be alternatively set through an input value, "ReferenceArea" (see AIM Inputs)
  • capsReferenceChord and capsReferenceSpan [Optional] These attribute may exist on any Body. Their value will be used as the reference moment lengths in ZAero's input file with their units assumed to be consistent with the attribute "capsLength". No conversion takes place if "capsLength" isn't set. These values may be alternatively set through an input value, "Moment_Length" (see AIM Inputs)
  • capsReferenceX, capsReferenceY, and capsReferenceZ [Optional] These attribute may exist on any Body. Their value will be used as the center of gravity (CG) location in ZAero's input file with their units assumed to be consistent with the attribute "capsLength". No conversion takes place if "capsLength" isn't set. These values may be alternatively set through an input value, "Moment_Center" (see AIM Inputs)
